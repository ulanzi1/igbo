## Challenges

1. **Hardcoded strings in 3/4 stories — locale variant not covered by AI-3** — Despite AI-3 (i18n keys in Task 1), hardcoded English strings appeared at HIGH severity in 7.1 (6 strings in EventMembershipGate/EventDetailActions/EventForm) and hardcoded `"en"` locale in `Intl.DateTimeFormat` appeared at HIGH in both 7.2 and 7.3. The locale hardcoding is a different failure mode than missing i18n keys — AI-3 was correctly applied but didn't address this variant.

2. **Admin permission gate dead code (7.4)** — `preserveRecording()` had identical `event.creatorId !== userId` checks on both outer and inner conditions, making admin preservation impossible. Same category as Story 5.4 ban bypass — additive permission audit lesson not applied to 7.4.

3. **Mutation test coverage still slipping** — 7.1 M-4 (no event detail page test), 7.2 H4/H5 (RSVPButton mutation coverage + no `getAttendeeStatus` test) despite explicit Epic 6 callout on this pattern.

4. **Machine-to-machine CSRF gap** — `withApiHandler` applies CSRF Origin validation on all POSTs. Daily.co webhooks can't satisfy this. Story 7.4 webhook test used `withApiHandler` passthrough mock (documented anti-pattern from Story 4.3). No first-class solution exists yet.

5. **Recording smoke test incomplete** — Steps 4–8 (attended transition, recording, webhook, mirror, download) not yet completed. Recording feature gates on completing these before production launch.

---

## Key Technical Decisions & Lessons Learned

- **Action items fix the observed pattern, not the class of problem.** AI-3 fixed missing i18n keys — but locale hardcoding (`"en"` in `Intl.DateTimeFormat`) is a different failure mode. Future action items must generalize to the class of problem.
- **Pre-epic spikes deliver.** AI-6 (Daily.co spike) → 7.3 implemented correctly first attempt. Redis+Lua needs the same treatment before Epic 8.
- **Admin permission paths need explicit regression tests every time.** The `preserveRecording` dead-code gate is the second admin-bypass bug in three epics.
- **Machine-to-machine webhooks are a first-class architecture concern.** `skipCsrf` option for `withApiHandler` is now a concrete need.
- **DAILY_API_KEY/DAILY_WEBHOOK_SECRET must be in staging before recording can be tested.** Core video path confirmed; recording tail requires complete end-to-end smoke test.
- **Known tech debt deferred:** `streamToBuffer()` in recording-mirror (RAM load for large recordings); S3 key reconstruction via regex in `getRecordingDownloadUrl`. Both acceptable for MVP.
