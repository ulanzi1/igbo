## What Went Well

1. **MessageService abstraction** — `PlaintextMessageService` implements a clean interface; E2E encryption is a well-defined seam for future epics. Kept API routes, services, and socket handlers independently testable.

2. **Two-namespace Socket.IO design stayed clean** — `/notifications` and `/chat` namespaces had no scope creep across 7 stories. Redis adapter for presence and typing state kept the app server stateless.

3. **Step-by-step TDD discipline** — Implementing service layer → API routes → Socket.IO handlers → UI, with tests at each layer, caught integration issues early. Story 2.5's three production bugs (ACK shape mismatch, rollback snapshot bug, content-blanking) were caught during TDD, not in production.

4. **Testing patterns codified** — `vi.hoisted()` for mutable mock context, `vi.useRealTimers()` for React Query + `waitFor` — discovered the hard way but now documented and reusable.

5. **Full epic delivered** — 7/7 stories done. Members can find each other, start conversations, share files, thread discussions, search messages, and manage presence in real-time. Core platform value proposition is live.

6. **TanStack Query + Socket.IO prep paid off** — Optimistic update with pre-snapshot rollback pattern landed cleanly in Stories 2.4 and 2.5 because of the Epic 1 retro groundwork.

7. **Test count trajectory** — 973 → 1,640 (+667 tests, ~95 per story average). Coverage was not sacrificed to ship faster.

## Challenges

1. **Story 2.3 scope and review volume** — 34 review findings (highest of either epic). Story combined group creation, member search, member management, system messages, and socket room coordination — too many mixed concerns in one story.

2. **i18n violations in 5/7 stories** — Persistent despite being Epic 1's #1 lesson. Pattern: UI gets built first, i18n added under review pressure. "Add it later" mindset must be eliminated.

3. **`waitFor` + fake timers hang** — RTL's `waitFor` uses `setInterval` for polling; `vi.useFakeTimers()` freezes it indefinitely. Discovery cost hours; symptom (infinite hang) is misleading. Fix: `vi.useRealTimers()` for any test involving React Query data fetching.

4. **Bug discovery burden on Dev** — Several correctness issues (joined_at message visibility, ACK shape mismatch, block enforcement gaps) only surfaced through Dev's manual testing. Unit tests written against the same incorrect assumption miss these.

5. **Real-time edge case coverage gap** — No integration-level tests for Socket.IO coordination layer. Out-of-order events, Redis TTL expiry mid-session, reconnect state — these are high-risk scenarios with no automated coverage.

6. **Deferred scope carries future bug risk** — Desktop overlay, group unread filtering by joined_at, `useUnreadCount` API hydration all deferred. Will surface as user-reported bugs once the platform has active members.

7. **Story 2.4 Dev Agent Record incomplete** — Story closed with task checklist unchecked. Process gap in story closure audit trail.

## Team Agreements (Carried Forward + New)

From Epic 1 (still active):

- i18n-first: all user-facing strings via `useTranslations()` — no hardcoded strings _(reinforced as AI-1)_
- EventBus from services only — never from API routes
- `server-only` imports enforced at service layer
- Co-located tests (not `__tests__` directories)
- No barrel exports from `src/db/schema/`

New from Epic 2:

- i18n keys defined in `en.json` + `ig.json` **before** any UI component is scaffolded
- `vi.useRealTimers()` required for tests involving React Query `waitFor` — do not use `vi.useFakeTimers()` in these contexts
- `vi.hoisted()` required for mutable mock context shared across test cases
- CSRF `Origin` header required in PATCH/DELETE route tests
- Zod v4: `parsed.error.issues[0]` (not `parsed.issues[0]`)
- URL parsing for nested routes: `.at(-2)` for `/{resource}/{id}/{action}` patterns
- Story closure requires Dev Agent Record checklist fully resolved
